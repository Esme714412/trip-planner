# 旅行計畫助手 Trip Planner

一個基於 React + Firebase 的旅遊行程管理 Web App，支援多人共享、即時同步、記帳分帳。

---

## 技術架構

- **前端**：React + Vite + Tailwind CSS
- **後端**：Firebase Authentication（Google 登入）+ Firestore（即時資料庫）
- **部署**：GitHub Pages（GitHub Actions 自動部署）

---

## Firestore 資料結構

```
users/{uid}/trips/{tripId}        # 自己的行程
sharedTrips/{tripId}              # 共享行程索引（editors[], viewers[]）
userProfiles/{uid}                # 使用者資料（email, displayName, nickname）
allowedUsers/{email}              # 白名單
```

## 環境設定

```bash
npm install
npm run dev
```

Firebase 設定透過環境變數注入，請在根目錄建立 `.env`：

```
VITE_FIREBASE_API_KEY=
VITE_FIREBASE_AUTH_DOMAIN=
VITE_FIREBASE_PROJECT_ID=
VITE_FIREBASE_STORAGE_BUCKET=
VITE_FIREBASE_MESSAGING_SENDER_ID=
VITE_FIREBASE_APP_ID=
```

GitHub Actions 部署時需在 repo Settings → Secrets and variables → Actions 設定對應的 Secrets。

## Markdown 行程格式

設定頁可**匯出**目前行程為 Markdown，也可**匯入** Markdown 新增行程。匯出的格式可直接貼給 AI（如 Claude、ChatGPT）請它規劃或修改行程。

```markdown
# 京阪古都之旅
- 日期：2026-05-01 ~ 2026-05-09
- 幣別：TWD

## 必帶物品
- 護照
- 換洗衣物（9天份）
- 行動電源

## 住宿
- 入住：2026-05-01
  退房：2026-05-03
  名稱：APA Hotel 道頓堀
  地點：大阪市中央區

## 行程

### 2026-05-01

#### [景點] 01:45 桃園機場出發
- 地點：桃園國際機場第一航廈
- 備註：提前2小時抵達，辦理出境

#### [交通] 03:30
- 交通方式：樂桃航空
- 搭車地點：桃園國際機場
- 下車地點：關西國際機場
- 預估時間：3小時
- 票價：6260
- 需提前購票：是
- 購票截止：出發前60天
- 購票連結：https://example.com

#### [景點] 09:30 抵達關西國際機場
- 地點：關西國際機場第2航廈
- 備註：領行李、換 ICOCA

## 收藏景點
- 名稱：錦市場
  備註：京都著名食材市場，適合早上逛
  連結：https://example.com
```

**欄位說明：**
- `購票截止` 支援 `YYYY-MM-DD` 或 `出發前N天`（自動換算）
- `## 必帶物品` 下的項目匯入後自動加入行前清單
- 交通 `需提前購票：是` 匯入後自動建立購票提醒清單項目

**給 AI 的 prompt 建議：**
```
請用以下 Markdown 格式整理行程，我會直接匯入旅遊規劃 App：
[貼上匯出的 Markdown 內容]

請根據以上行程幫我：[你的需求]
```
