# 旅行計畫助手 Trip Planner

一個基於 React + Firebase 的旅遊行程管理 Web App，支援多人共享、即時同步、記帳分帳。

---

## 技術架構

- **前端**：React + Vite + Tailwind CSS
- **後端**：Firebase Authentication（Google 登入）+ Firestore（即時資料庫）
- **部署**：GitHub Pages（GitHub Actions 自動部署）

---

## Firestore 資料結構

```
users/{uid}/trips/{tripId}        # 自己的行程
sharedTrips/{tripId}              # 共享行程索引（editors[], viewers[]）
userProfiles/{uid}                # 使用者資料（email, displayName, nickname）
allowedUsers/{email}              # 白名單
```

---

## 環境設定

```bash
npm install
npm run dev
```

Firebase 設定透過環境變數注入，請在根目錄建立 `.env`：

```
VITE_FIREBASE_API_KEY=
VITE_FIREBASE_AUTH_DOMAIN=
VITE_FIREBASE_PROJECT_ID=
VITE_FIREBASE_STORAGE_BUCKET=
VITE_FIREBASE_MESSAGING_SENDER_ID=
VITE_FIREBASE_APP_ID=
```

GitHub Actions 部署時需在 repo Settings → Secrets and variables → Actions 設定對應的 Secrets。

## Markdown 匯入格式

```markdown
# 旅程名稱（選填）

## 2025-05-01
- time: 09:00
  title: 關西國際機場          # 必填
  location: 關西國際機場
  transport: 關空特急 Haruka
  transport_departure: 09:30
  transport_duration: 75分鐘
  transport_price: 3600
  transport_url: https://www.westjr.co.jp/
  transport_ticket: true       # true = 需提前購票，自動加入行前清單
  hours: 09:00-22:00
  website: https://...
  notes: 注意事項備註
  tickets: 門票資訊
  shopping:
    - 購物項目

## 行前清單
- [ ] 未完成事項
- [x] 已完成事項
```

**注意**：
- 只有 `title` 為必填，其餘欄位皆可省略
- 日期格式：`YYYY-MM-DD`
- 匯入的內容會附加到現有行程後面，不會覆蓋
